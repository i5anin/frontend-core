### **SPA**

**Суть:**

- Приложение загружается один раз.
    
- Навигация и рендеринг выполняются на клиенте.
    
- Сервер отдаёт один HTML и набор JS-бандлов.
    

**Плюсы:**

- Быстрые переходы.
    
- Богатая интерактивность.
    

**Минусы:**

- Медленный первый рендер.
    
- Зависимость от JS.
    

---

### **CSR**

**Суть:**

- Весь рендеринг выполняется в браузере.
    
- Сервер отдаёт «пустой» HTML + JS.
    
- Данные подтягиваются через API.
    

**Плюсы:**

- Гибкость, полный контроль на клиенте.
    

**Минусы:**

- Плохое SEO.
    
- Долгий TTFB → FCP.
    

---

### **SSR**

**Суть:**

- Сервер рендерит HTML на основе данных.
    
- Клиент получает уже готовую страницу.
    
- Затем происходит гидратация.
    

**Плюсы:**

- Быстрый первый экран.
    
- Хорошее SEO.
    

**Минусы:**

- Высокая нагрузка на сервер.
    
- Сложнее инфраструктура.
    

---

### **SSG**

**Суть:**

- HTML генерируется на этапе сборки.
    
- Готовые страницы отдаются как статика.
    
- Подходит для контентных сайтов.
    

**Плюсы:**

- Максимальная скорость.
    
- Минимальная нагрузка.
    

**Минусы:**

- Нет динамики без доп. API.
    
- Нельзя обновлять контент мгновенно.
