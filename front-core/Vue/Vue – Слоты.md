Слоты — механизм передачи разметки от родительского компонента в дочерний. Они позволяют создавать гибкие и переиспользуемые UI-компоненты, не жёстко задавая внутреннюю структуру.

**Базовый слот**

```vue
<!-- Родитель -->
<Card>
  <p>Контент карточки</p>
</Card>

<!-- Дочерний компонент -->
<template>
  <div class="card">
    <slot></slot>
  </div>
</template>
```

Если родитель ничего не передал, может быть дефолтное содержимое:

```vue
<slot>Пусто</slot>
```

**Именованные слоты**

Используются, когда компонент должен принимать несколько зон разметки.

```vue
<Card>
  <template #header>Заголовок</template>
  <template #footer>Футер</template>
</Card>
```

Дочерний компонент:

```vue
<div class="card">
  <header><slot name="header"></slot></header>
  <main><slot></slot></main>
  <footer><slot name="footer"></slot></footer>
</div>
```

**Слоты со свойствами (scoped slots)**  
Позволяют дочернему компоненту передавать данные обратно в разметку родителя.

Дочерний:

```vue
<slot :item="user"></slot>
```

Родитель:

```vue
<Card v-slot="{ item }">
  <p>{{ item.name }}</p>
</Card>
```

Суть: дочерний компонент отдаёт данные, а родитель сам решает, как их визуализировать.

**Особенности поведения**

- Слоты рендерятся в контексте родителя — доступ к переменным идёт из родителя.
    
- Внутренний компонент решает только структуру контейнеров, а содержимое задаёт родитель.
    
- Scoped slots позволяют строить полностью «headless» компоненты — компонент поставляет данные и логику, родитель обеспечивает визуализацию.
    

**Использование**

- UI-компоненты: карточки, списки, модалки, таблицы.
    
- Headless-компоненты: выпадающие меню, селекты, таблицы с управляемым рендером.
    
- Гибкая кастомизация внешнего вида без изменения компонента.
    

Слоты — фундаментальный механизм Vue, обеспечивающий расширяемость и глубоко настраиваемую структуру пользовательских интерфейсов.
