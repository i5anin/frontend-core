Options API — классический стиль разработки во Vue, в котором логика компонента разделена по секциям: `data`, `methods`, `computed`, `watch`, хуки жизненного цикла. Каждая секция отвечает за строго определённый тип данных или поведения.

**Структура компонента**

```js
export default {
  data() {
    return {
      count: 0
    }
  },
  computed: {
    double() {
      return this.count * 2
    }
  },
  methods: {
    increment() {
      this.count++
    }
  },
  watch: {
    count(value) {
      console.log('changed', value)
    }
  },
  mounted() {
    console.log('mounted')
  }
}
```

**Ключевые особенности**

- Основан на объекте-конфигурации.
    
- Все свойства доступны через `this` — это экземпляр компонента.
    
- Данные, вычисляемые свойства, методы и наблюдатели разделены по разным разделам — логика распределяется вертикально.
    
- Отлично подходит для небольших компонентов и простых UI-задач.
    
- Имеет ясную структуру и низкий порог входа.
    

**Хуки жизненного цикла**

Основные:  
`created`, `mounted`, `updated`, `unmounted`.  
В Options API все хуки — методы объекта.

**Ограничения и недостатки**

- При усложнении компонента логика расползается по секциям — трудно воспринимать.
    
- `this` должен быть корректно типизирован, что усложняет работу с TypeScript.
    
- Код менее удобен для повторного использования, чем Composition API.
    

Options API остаётся поддерживаемым и стабильным стилем, но в современных проектах чаще используется как базовый уровень или для простых компонентов.
