Псевдоклассы описывают **состояние элемента**, которое нельзя выразить обычными селекторами. Основные:

`:hover` — наведение курсора  
`:active` — нажатие  
`:focus` — фокус ввода  
`:disabled` — неактивный элемент формы  
`:checked` — выбранный checkbox/radio  
`:required` / `:optional` — состояние форм  
`:valid` / `:invalid` — результат валидации  
`:first-child`, `:last-child` — позиция в дереве  
`:nth-child(n)` — выбор по порядку  
`:not(selector)` — отрицание  
`:target` — элемент с id из hash  
`:visited` / `:link` — состояние ссылки

Псевдоэлементы создают **виртуальные элементы**, которых нет в DOM, но есть в CSS.

Основные:

`::before` / `::after` — вставка контента перед/после элемента. Работают только при наличии `content`.  
`::placeholder` — стиль текста placeholder в инпутах.  
`::selection` — стиль выделенного пользователем текста.  
`::first-line` / `::first-letter` — оформление первой строки/буквы.  
`::marker` — маркеры списков.  
`::backdrop` — фон модальных элементов (`dialog`, fullscreen).

Ключевые свойства поведения:

- Псевдоэлемент — часть реального элемента, но рендерится отдельно.
    
- `::before`/`::after` не вставляют HTML, только CSS-контент.
    
- Псевдоклассы не создают новые элементы — они переключают стиль существующего.
    
- Специфичность псевдоклассов равна классу; псевдоэлементы тоже считаются как класс.
    
- Псевдоэлементы обычно располагаются поверх/под элементом, но участвуют в stacking context.
    

Эти механики используются для интерактивных состояний, декоративных элементов, кастомизации форм и построения UI без лишнего HTML.
