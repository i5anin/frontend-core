БЭМ — методология именования и организации CSS, которая делает интерфейс предсказуемым, масштабируемым и изолированным. Основа — разделение компонента на три части: блок, элемент, модификатор.

**Блок**  
Независимая сущность интерфейса.  
Имя отражает назначение, не структуру и не внешний вид.  
Пример: `header`, `menu`, `card`.

```html
<div class="card"></div>
```

**Элемент**  
Часть блока, которая не существует отдельно.  
Записывается через двойное подчёркивание:

```html
<div class="card__title"></div>
<div class="card__content"></div>
```

Элемент всегда принадлежит только одному блоку. Вложенность в DOM не влияет на имя.

**Модификатор**  
Описывает состояние или вариант блока/элемента.  
Записывается через двойной дефис:

```html
<div class="card card--active">
<div class="card__button card__button--large">
```

Модификатор не меняет логику структуры — только внешний вид или состояние.

**Правила методологии**

- Уровень вложенности в CSS должен быть минимальным — имена уже гарантируют изоляцию.
    
- Внутри одного блока не используют селекторы вида `.card .other` — блоки не должны зависеть друг от друга.
    
- Элемент нельзя использовать без блока (`__title` допустим только как `card__title`).
    
- Модификаторы не меняют HTML-структуру.
    

**Зачем используется**

- Избегание конфликтов стилей.
    
- Предсказуемая структура классов.
    
- Удобная работа с компонентами в UI-библиотеках и SPA-фреймворках.
    
- Масштабируемость проекта: любой блок легко переносится в другой проект.
    

БЭМ — стабильная методология, которая помогает держать CSS чистым, изолированным и читаемым в большом коде.
