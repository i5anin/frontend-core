Изоляция стилей — это набор техник, которые предотвращают «протекание» CSS между компонентами и гарантируют, что оформление одного блока не ломает другой. В современных интерфейсах это обязательное требование для масштабируемости.

**БЭМ**  
Имена по строгой структуре (`block`, `block__element`, `block--mod`).  
Изоляция достигается за счёт уникальных пространств имён и отсутствия вложенности.

**CSS Modules**  
Каждый класс превращается в уникальное имя на этапе сборки (`button__3fa2a`).  
Стиль действует только внутри файла.  
Используется с Webpack/Vite.

**Scoped стили (Vue SFC)**  
Тег `<style scoped>` генерирует уникальные атрибуты (`data-v-xxxx`) и применяет CSS только к текущему компоненту.  
Селекторы автоматически переписываются.

**Shadow DOM**  
Создаёт полностью независимую область стилей.  
Внешние стили не влияют на компонент, и наоборот.  
Используется в Web Components.

**CSS-in-JS**  
Стили генерируются программно:  
Styled Components, Emotion, Vue CSS-in-JS решения.  
Каждый компонент имеет собственный скоуп стилей и уникальные классы.

**Изоляция через архитектуру**  
Минимизация глобальных стилей.  
Глобально допускаются только reset/normalize, variables, базовая типографика.  
Остальное оформляется на уровне компонентов.

**Почему это важно**  
Гарантируется отсутствие конфликтов между частями интерфейса.  
Повышается предсказуемость поведения UI.  
Компоненты становятся переносимыми.  
Технический долг в CSS не накапливается.

Изоляция стилей — фундамент для стабильной поддержки крупных проектов и компонентных архитектур.
