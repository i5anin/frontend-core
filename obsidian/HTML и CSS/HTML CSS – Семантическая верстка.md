Семантическая вёрстка — это использование HTML-тегов по их прямому назначению, чтобы структура страницы отражала смысл содержимого. Семантика улучшает доступность, SEO, поведение поисковиков и читаемость кода.

**Базовые семантические теги**  
`<header>` — шапка страницы или секции.  
`<nav>` — навигация.  
`<main>` — основное содержимое документа.  
`<section>` — смысловой раздел.  
`<article>` — независимый блок контента (публикация, карточка, статья).  
`<aside>` — боковая информация: сайдбар, вспомогательный контент.  
`<footer>` — подвал страницы или секции.  
`<figure>` и `<figcaption>` — медиа + подпись.  
`<time>` — временные данные.  
`<address>` — контактная информация.  
`<table>`, `<thead>`, `<tbody>`, `<th>` — табличные данные.  
`<form>` — логическая форма, а не контейнер «для удобства».

**Принципы применения**

- Структура документа должна отражать логику контента, а не визуальное расположение.
    
- Каждый блок выбирается по смыслу: «что это за информация?», а не «как она выглядит?».
    
- Заголовки `<h1–h6>` должны образовывать иерархию: последовательную и логичную.
    
- Один `<h1>` на страницу — основной заголовок документа.
    
- Не использовать `<div>` там, где подходит семантический элемент.
    

**Роль в доступности и SEO**

- Семантика улучшает работу скринридеров.
    
- Поисковики лучше понимают структуру документа.
    
- Правильные теги позволяют корректно формировать сниппеты и контекстное содержание.
    
- Семантические зоны помогают браузеру и поисковику классифицировать контент.
    

**Типовые ошибки**

- Использование `<div>` для навигации вместо `<nav>`.
    
- Использование заголовков неправильного уровня (прыжки: h1 → h4 → h2).
    
- Обёртка всей страницы в `<section>` без семантической необходимости.
    
- Множественные `<h1>` внутри одного документа без логического обоснования.
    

Семантическая вёрстка делает структуру HTML понятной для браузера, поисковиков и разработчиков, обеспечивая корректное поведение интерфейса и упрощая поддержку проекта.
