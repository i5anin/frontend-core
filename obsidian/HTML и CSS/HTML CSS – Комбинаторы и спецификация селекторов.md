### Комбинаторы CSS

| Комбинатор | Синтаксис       | Описание                                   | Пример                  | Выбирает                                      |
|------------|-----------------|--------------------------------------------|-------------------------|-----------------------------------------------|
| Дочерний   | `A > B`         | Прямые дочерние элементы                   | `ul > li`               | Только li, являющиеся прямыми детьми ul       |
| Смежный    | `A + B`         | Элемент, идущий сразу после A (один уровень)| `h2 + p`                | Первый p сразу после h2                       |
| Общий      | `A ~ B`         | Все элементы B, идущие после A (один уровень)| `h2 ~ p`              | Все p после h2 на том же уровне              |
| Потомок    | `A B` (пробел)  | Все B внутри A (любая вложенность)         | `div p`                 | Все p внутри div                              |

### Основные типы селекторов (по возрастанию специфичности)

| Тип              | Пример                | Специфичность (a-b-c-d) | Примечание                              |
|------------------|-----------------------|--------------------------|-----------------------------------------|
| Тип (элемент)    | `div`, `p`            | 0-0-0-1                  |                                         |
| Класс            | `.btn`                | 0-0-1-0                  |                                         |
| Атрибут          | `[type="text"]`       | 0-0-1-0                  |                                         |
| Псевдокласс      | `:hover`, `:nth-child`| 0-0-1-0                  |                                         |
| Псевдоэлемент    | `::before`            | 0-0-0-1                  |                                         |
| ID               | `#header`             | 0-1-0-0                  |                                         |
| Инлайн стиль     | `style=""`            | 1-0-0-0                  |                                         |
| `!important`     | `color: red !important`| побеждает всё          |                                         |

### Логические комбинаторы (CSS Selectors Level 4, 2025 — полная поддержка)

| Комбинатор       | Синтаксис                  | Описание                                          | Пример                          |
|------------------|----------------------------|---------------------------------------------------|---------------------------------|
| `:is()`          | `:is(A, B, C)`             | Любой из списка (специфичность = самый высокий)   | `:is(h1, h2, h3):hover`         |
| `:where()`       | `:where(A, B, C)`          | Любой из списка (специфичность = 0)               | `:where(section, article) p`    |
| `:not()`         | `:not(A, B)`               | Исключение (можно несколько)                      | `:not(.btn, .link):hover`       |
| `:has()`         | `A:has(B)`                 | A, содержащий B (родственник)                     | `section:has(h2)` — section с h2|

### Полезные современные селекторы (2025)

| Селектор               | Описание                                               | Пример                          |
|------------------------|--------------------------------------------------------|---------------------------------|
| `:nth-child(an+b of S)`| n-ый ребёнок среди отфильтрованных S                  | `:nth-child(odd of .item)`      |
| `:focus-visible`       | Фокус только при клавиатурной навигации                | `button:focus-visible`          |
| `:focus-within`        | Родитель элемента в фокусе                             | `form:focus-within`             |
| `:target`              | Элемент, на который указывает якорь URL                | `:target`                       |
| `:modal`               | Элемент `<dialog>` в открытом состоянии                | `dialog:modal`                  |
| `:picture` / `:video`  | Нативные медиа-элементы в состоянии воспроизведения    | `video:playing`                 |

### Порядок перечисления селекторов в правилах
Последнее правило побеждает при равной специфичности → важен порядок в файле.

### Ключевые моменты для собеседования
- Специфичность считается только по выигрышному селектору в сложных комбинаторах (кроме `:is()` — берёт максимум, `:where()` — 0).
- `:has()` не работает в старых браузерах, но с 2023–2025 поддерживается везде.
- Комбинаторы > и + ~ работают только между простыми селекторами, внутри `:is()`/:where()`/:not()` можно вкладывать.
