### Canvas и SVG — базовое сравнение и факты для собеседования 2025

| Характеристика              | **SVG** (Scalable Vector Graphics)                          | **HTML Canvas** (2D / WebGL)                                      |
|-----------------------------|-------------------------------------------------------------|-------------------------------------------------------------------|
| Тип                         | Векторный формат, DOM-элемент                               | Растровый буфер, рисуется скриптом                                |
| Структура                   | XML-подобный код, каждый объект — отдельный DOM-узел        | Один `<canvas>` элемент, всё рисуется через JS                     |
| Разрешение                  | Бесконечное масштабирование без потери качества             | Пиксельный, масштабирование = размытие/пикселизация               |
| Производительность          | Низкая при > 1000–2000 сложных элементов                   | Высокая при тысячах объектов (особенно с `offscreenCanvas`)      |
| Интерактивность             | Нативная (события на каждый `<path>`, `<circle>` и т.д.)    | Только на весь canvas (нужно самостоятельно определять попадания)|
| Доступность                 | Полная (текст, `aria-label`, читается скринридерами)        | Практически отсутствует (нужен fallback или aria-описание)       |
| Анимация                    | CSS / SMIL / JS (высокая производительность)               | Только JS (requestAnimationFrame)                                |
| Поиск / копирование текста  | Есть (текст остаётся текстом)                               | Нет (текст превращается в пиксели)                                |
| Поддержка браузерами 2025   | 100% с 2011 года                                            | 100% + WebGL 2.0 / WebGPU в стадии широкого внедрения             |

### Когда выбирать SVG (2025)

- Логотипы, иконки, инфографика  
- Диаграммы, графики (D3.js, Chart.js с SVG-режимом)  
- Интерактивные карты  
- Нужно масштабирование + доступность + SEO  
- Количество объектов ≤ 2000  

### Когда выбирать Canvas

- Игры, сложные визуализации (Three.js, PixiJS, Konva.js)  
- Обработка изображений в реальном времени  
- Графики с > 10 000 точек (например, финансовые свечи)  
- WebGL / WebGPU 3D-сцены  
- Нужна максимальная производительность  

### Ключевые методы Canvas 2D (2025 — самые используемые)

```js
const ctx = canvas.getContext('2d');

// Основные
ctx.fillStyle / strokeStyle
ctx.fillRect(x, y, w, h)
ctx.strokeRect(x, y, w, h)
ctx.clearRect(x, y, w, h)

// Пути
ctx.beginPath()
ctx.moveTo / lineTo / arc / bezierCurveTo
ctx.closePath()
ctx.fill() / stroke()

// Текст
ctx.font = '48px serif'
ctx.fillText('Hello', x, y)
ctx.strokeText()

// Изображения
ctx.drawImage(img, dx, dy)                     // простое
ctx.drawImage(img, sx, sy, sw, sh, dx, dy, dw, dh)  // кроп + масштаб

// Современные 2025
ctx.getContext('2d', { alpha: false, desynchronized: true }) // производительность
OffscreenCanvas + worker // перенос тяжёлых вычислений
ctx.filter = 'blur(4px) drop-shadow(...)' // CSS-фильтры в canvas
```

### SVG — базовые элементы и атрибуты 2025

```xml
<svg viewBox="0 0 100 100" xmlns="http://www.w3.org/2000/svg">
  <circle cx="50" cy="50" r="40" fill="red"/>
  <path d="M10,30 A20,20 0,0,1 50,30" stroke="blue" fill="none"/>
  <text x="10" y="20" font-size="16">Текст</text>
  <use href="#mySymbol"/>           <!-- повторное использование -->
  <mask>, <clipPath>, <filter>      <!-- эффекты -->
</svg>
```

### Важные современные возможности SVG (2025)

- `<svg>` как `<img src="icon.svg">` — полная поддержка  
- `href` вместо устаревшего `xlink:href`  
- CSS-переменные внутри SVG  
- `viewBox` + `preserveAspectRatio` — полный контроль масштабирования  
- `<foreignObject>` — вставка HTML внутрь SVG  

### Вывод для собеседования (коротко)

| Задача                            | Выбор 2025          |
|-----------------------------------|---------------------|
| Иконки / логотип                  | SVG                 |
| Сложная интерактивная диаграмма   | SVG (до 2–3k элементов) |
| Игра / 3D                         | Canvas + WebGL/WebGPU |
| График 50 000 точек               | Canvas              |
| Нужно SEO и доступность           | SVG                 |
| Максимальная производительность   | Canvas + OffscreenCanvas |

**Главное правило 2025:**  
SVG — для контента и простых интерактивных графиков.  
Canvas — для игр, тяжёлых визуализаций и всего, что требует 60 fps при тысячах объектов.
