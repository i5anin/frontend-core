Формы используются для ввода и отправки данных. Основная структура задаётся тегом `<form>` с атрибутами `action`, `method`, `autocomplete`, `novalidate`.

Ключевые элементы формы:

`<input>` — универсальное поле. Основные типы:  
`text`, `password`, `email`, `number`, `tel`, `url`, `date`, `time`, `file`, `checkbox`, `radio`, `range`, `hidden`, `search`.  
Отдельные особенности:

- `checkbox` и `radio` имеют состояние `checked`.
    
- `file` возвращает `FileList`.
    
- `number` имеет встроенные ограничения: `min`, `max`, `step`.
    

`<textarea>` — многострочный ввод текста. Поддерживает `maxlength`, `placeholder`, `resize`.

`<select>` — выпадающий список.  
`<option>` — варианты.  
`<optgroup>` — группировка вариантов.

`<button>` — кнопка.  
Типы: `submit`, `reset`, `button`. Без `type` по умолчанию — `submit`.

`<label>` — связывается с полем через `for` или обёртку. Улучшает доступность и увеличивает зону клика.

`<fieldset>` — группировка связанных полей.  
`<legend>` — название группы.

Валидация встроенными механизмами HTML:

- `required` — обязательное поле
    
- `min`, `max`, `step` — числовые поля
    
- `pattern` — регулярное выражение
    
- `minlength`, `maxlength` — длина строки
    
- `type=email/url` — базовая проверка формата
    

События:  
`input` — изменение значения без потери фокуса  
`change` — изменение и подтверждение действия  
`submit` — отправка формы  
`focus` / `blur` — фокусировка/потеря

Особенности поведения:

- Отправка формы по кнопке `submit` или по Enter внутри текстового поля.
    
- Браузер может заблокировать отправку при невалидных данных (HTML5 Validation API).
    
- Доступ к данным формы обеспечивается через `FormData` в JS.
    
- Для кастомного UI почти всегда скрывают реальные `checkbox/radio` и управляют стилями через псевдоэлементы.
    

Формы требуют продуманной доступности: связь `label + input`, корректные типы, порядок фокуса (`tabindex`), читаемая структура.
