Встраивание медиа — это подключение изображений, видео и аудио в HTML-документ с контролем отображения, форматов и поведения.

**Изображения**  
Основной элемент — `<img>`.  
Ключевые атрибуты:  
`src` — путь к файлу  
`alt` — текстовое описание (обязателен для доступности)  
`width` / `height` — размеры  
`loading="lazy"` — отложенная загрузка  
`srcset` и `sizes` — адаптивные изображения

Пример адаптивной загрузки:

```html
<img 
  src="small.jpg"
  srcset="small.jpg 480w, medium.jpg 800w, large.jpg 1200w"
  sizes="100vw"
  alt="Фото"
/>
```

Для сложных кейсов используется `<picture>` — выбор формата/разрешения в зависимости от условий.

---

**Видео**  
Элемент `<video>`.

Основные атрибуты:  
`controls` — отображение стандартных контролов  
`autoplay` — автозапуск (часто требует `muted`)  
`loop`  
`muted`  
`poster` — отображается перед стартом  
`preload` — стратегия загрузки

Пример:

```html
<video controls width="600" poster="preview.jpg">
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
</video>
```

Форматы: `.mp4`, `.webm`, `.ogg`.

---

**Аудио**  
Элемент `<audio>`.

Используемые атрибуты:  
`controls`, `autoplay`, `loop`, `preload`.

```html
<audio controls>
  <source src="sound.mp3" type="audio/mpeg">
  <source src="sound.ogg" type="audio/ogg">
</audio>
```

---

**Медиа через CSS**  
Фоновые изображения задаются через `background-image`.  
Поддерживаются современные форматы (`webp`, `avif`).

---

**Глубокие особенности:**

- Браузер выбирает первый подходящий `<source>` по MIME-типу.
    
- `<video>` и `<audio>` не блокируют построение DOM, но могут задерживать полноценный рендер, если `preload="auto"`.
    
- Правильный `alt` обязателен — обеспечивает доступность и fallback.
    
- Использование `srcset` снижает трафик и повышает скорость загрузки.
    
- Видео без `muted` не запускается автоматически в современных браузерах.
    

Встраивание медиа — ключевой блок, влияющий на доступность, производительность и адаптивность интерфейса.
