Компоненты во Vue — это изолированные и переиспользуемые единицы интерфейса. Каждый компонент инкапсулирует разметку, логику и стили (в пределах SFC), формируя основу для масштабирования приложения.

**Структура SFC (Single File Component)**

```vue
<template>
  <button @click="increment">{{ count }}</button>
</template>

<script setup>
import { ref } from 'vue'

const count = ref(0)
const increment = () => count.value++
</script>

<style scoped>
button { padding: 8px 12px; }
</style>
```

SFC объединяет всё, что относится к компоненту:

- шаблон,
    
- логика (Composition API),
    
- локальные стили (`scoped`).
    

**Свойства компонента**

`props` — входные параметры:

```js
defineProps({
  label: String,
  disabled: Boolean
})
```

`emit` — события, которые компонент может отправлять наружу:

```js
const emit = defineEmits(['submit'])
emit('submit', payload)
```

`slots` — передача разметки от родителя:

```html
<CustomCard>
  <div>Контент</div>
</CustomCard>
```

`expose()` — управляемая передача методов родителю, когда нужно контролируемое взаимодействие.

**Изоляция и повторное использование**

Компоненты:

- не зависят от окружения, кроме переданных props,
    
- содержат минимально возможное количество логики,
    
- не изменяют состояние родителя напрямую,
    
- не модифицируют DOM вне своей зоны ответственности.
    

**Жизненный цикл**

Компонент проходит этапы: создание → монтирование → обновление → размонтирование.  
В Composition API используются хуки: `onMounted`, `onUpdated`, `onUnmounted` и др.

**Компонентная модель**

- Верхнеуровневые компоненты для страниц.
    
- Промежуточные layout-компоненты.
    
- Малые UI-компоненты (кнопки, инпуты).
    
- Компоненты должны быть мелкими, однозначными и выполняющими одну задачу.
    

**Правила хорошего дизайна**

- Компонент не должен выполнять больше одной логической функции.
    
- Логику выносить в composables, а не раздувать SFC.
    
- Интерфейс компонента = аккуратные props + события + стабильный контракт.
    
- Стили изолированы (scoped или CSS Modules).
    

Компоненты формируют архитектуру Vue-приложения, обеспечивая переиспользуемость, читаемость и аккуратное разделение ответственности.
