Provide/Inject — механизм передачи данных по иерархии компонентов **вниз**, минуя промежуточных родителей. Используется для глобальных зависимостей внутри ветки дерева и построения инфраструктурных компонентов.

**Provide (в родителе)**  
Родитель объявляет значения, доступные для вложенных компонентов:

```vue
<script setup>
import { provide, ref } from 'vue'

const theme = ref('dark')
provide('theme', theme)
</script>
```

**Inject (в потомке)**  
Любой вложенный компонент может получить значение:

```vue
<script setup>
import { inject } from 'vue'

const theme = inject('theme')
</script>
```

`theme` здесь — та же самая реактивная ссылка, управление идёт сверху.

**Особенности и важные детали**

- Связь **не реактивная сама по себе**, но если передаётся реактивное значение (`ref`, `reactive`), оно будет обновляться.
    
- Inject видит только то, что было provide'нуто вверх по цепочке.
    
- Provide не зависит от глубины: потомок может быть хоть через 10 уровней.
    
- Использование — **однонаправленное**, потомок не должен менять предоставленные данные напрямую без контроля.
    

**Типичные сценарии использования**

- Тема, локализация, конфигурация UI.
    
- Логика крупных компонентов: таблицы, формы, сложные виджеты.
    
- Реализация собственных плагинов.
    
- Расшаривание общих функций и сервисов внутри ветки компонентов.
    
- Контекстные данные (например, группировка инпутов в форме).
    

**Отличие от props**

- Props — явная передача родитель → ребёнок, строгий контракт.
    
- Provide/Inject — скрытая контекстная передача, без обязательного API.
    
- Props подходит для UI-компонентов; Provide/Inject — для инфраструктурных.
    

Provide/Inject помогает строить сложные архитектуры, разделять ответственность и передавать зависимости в глубину дерева без раздувания props.
