Render-функция во Vue — это альтернативный способ описывать разметку компонента с помощью чистого JavaScript вместо шаблона `<template>`. Она даёт полный контроль над тем, как создаётся виртуальный DOM.

**Базовая форма**

```js
import { h } from 'vue'

export default {
  render() {
    return h('button', { onClick: this.onClick }, this.label)
  }
}
```

`h` — это фабрика виртуальных узлов (VNode).  
Она принимает:

- тип (`'div'`, компонент, Fragment),
    
- props/атрибуты,
    
- дочерние элементы.
    

**Render-функция в `<script setup>`**

```vue
<script setup>
import { h } from 'vue'

const render = () => h('div', { class: 'box' }, 'text')
</script>

<template>
  <render />
</template>
```

**Когда используется**

- Сложные динамические разметки, которые тяжело выразить в шаблонах.
    
- Headless-компоненты, где шаблона нет, а рендер зависит от данных родителя.
    
- Создание UI-библиотек и кастомных компонентов низкого уровня.
    
- Программная генерация структур (например, таблицы, древовидные структуры).
    

**Работа со слотами**

Слоты передаются как функции:

```js
render() {
  return h('div', {}, this.$slots.default?.())
}
```

Scoped-слоты:

```js
h('div', {}, this.$slots.default?.({ item }))
```

**Преимущества**

- Полный контроль над виртуальным DOM.
    
- Нет ограничений синтаксиса шаблонов.
    
- Проще программно генерировать сложные структуры.
    
- Подходит для инфраструктурных компонентов.
    

**Недостатки**

- Менее читаемо для обычного UI-кода.
    
- Дольше писать.
    
- Сложнее понимать новичкам.
    

**Типовые решения на Render**

Программное создание списков:

```js
render() {
  return h('ul', {}, this.items.map(i => h('li', {}, i)))
}
```

Динамическая подмена компонента:

```js
render() {
  return h(this.isModalOpen ? Modal : Placeholder)
}
```

Render-функция — инструмент низкого уровня, который используют, когда шаблоны ограничивают. В обычных компонентах она не нужна, но в сложных UI-паттернах незаменима.
