## Функциональное программирование в `JavaScript` — краткая инструкция

### 1. Базовые принципы

- `Первая роль функций`: функции — данные, их можно передавать, возвращать, сохранять.
    
- `Чистые функции`: одинаковый ввод → одинаковый вывод, отсутствие побочных эффектов.
    
- `Иммутабельность`: исходные данные не изменяются, создаётся новое значение.
    
- `Декларативный стиль`: описание результата, а не пошаговой логики.
    

---

### 2. Функции высшего порядка

Функции, принимающие другие функции или возвращающие их.  
Примеры: `map`, `filter`, `reduce`.

---

### 3. Композиция функций

Объединение нескольких функций в цепочку.  
Пример простой композиции:

```js
const compose = (f, g) => x => f(g(x))
```

---

### 4. Каррирование

Преобразование функции в последовательность вызовов по одному аргументу.

```js
const add = a => b => a + b
```

---

### 5. Иммутабельные операции над структурами

- Работа через `spread`, `Object.assign`, `Array.map`, `Array.filter`.
    
- Запрет прямой модификации исходных объектов и массивов.
    

---

### 6. Рекурсия

В `FP` используется вместо циклов.  
Важно: корректная терминальная ветка для предотвращения переполнения стека.

---

### 7. Функции без состояния (`stateless`)

Логика не опирается на внешние изменяемые данные.  
Повышает предсказуемость и упрощает тестирование.

---

### 8. Частичное применение (`partial application`)

Фиксация части аргументов:

```js
const partial = (fn, ...a) => (...b) => fn(...a, ...b)
```

---

### 9. Работа с коллекциями через декларативные методы

- `map` — создание новой коллекции
    
- `filter` — отбор
    
- `reduce` — свёртка к одному значению
    
- `flatMap`, `find`, `every`, `some`
    

---

### 10. Ошибки, которые проверяют на собесе

- Путаница между `каррированием` и `частичным применением`.
    
- Использование мутаций в ответах.
    
- Неумение объяснить, что такое “чистая функция”.
    
- Отсутствие понимания, что `JavaScript` не является чистым `FP-языком`, но поддерживает ключевые механизмы.
