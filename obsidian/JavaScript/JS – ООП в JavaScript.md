## ООП в `JavaScript` — краткое описание

### 1. Модель

`JavaScript` реализует объектную модель на основе `прототипного наследования`, а не `классового` как в `Java`, `C#`.

### 2. Базовые механизмы

- `[[Prototype]]` — скрытая ссылка на объект-родитель.
    
- `Object.create(proto)` — создание объекта с заданным прототипом.
    
- Поиск свойств идёт по цепочке прототипов.
    

### 3. Синтаксис `class`

Надстройка над прототипами.  
Пример:

```js
class User {
  constructor(name) {
    this.name = name
  }

  getName() {
    return this.name
  }
}
```

### 4. Наследование

Ключевое слово `extends` создаёт связь прототипов.

```js
class Admin extends User {
  remove() {}
}
```

Внутри конструктора вызов базового:

```js
super()
```

### 5. Инкапсуляция

В `JavaScript` поддерживаются приватные поля через `#privateField`.

```js
class Account {
  #balance = 0
  getBalance() { return this.#balance }
}
```

### 6. Полиморфизм

Методы могут переопределяться в наследниках.

```js
class A { run() {} }
class B extends A { run() {} }
```

### 7. Абстракция

Через соглашения, т.к. строгих абстрактных классов нет.  
Частично реализуется через:

- базовые `class`
    
- приватные поля
    
- выброс ошибок в методах, требующих переопределения
